Problem #3

Write a public static function run in a class called Solution that takes two arguments, a BufferedReader from which to read inputs and a PrintWriter on which to write output, and which may throw an IOException.  You should use the skeleton provided under course documents on COL. You may optionally design/create other classes as well.  For this assignment, a class for building and traversing trees may be useful.

You will also need the test client which you can download from course documents on COL.

To run the test, make sure TestClient.java and your Solution.java are in the same directory and compile them.  You can then run the test at the command line with:

    java TestClient uname num

where uname consists of the first letter of your first name followed by the first seven letters of your last name, and num is the problem number.  For example, if David Letterman were submitting a solution to problem #3, he would type in

    java TestClient dletterm 3

at the command line.

You can check your grade at:  http://wmarrerowww.cstcis.cti.depaul.edu/csc421/problem3.html

Input:

Each line of input coming in on the BufferedReader represents a separate problem.  (Hint: Consider using the readLine() method in the BufferedReader class).  Each line consists of two strings of capital letters separated by a single space.  The first string represents a preorder traversal of a binary tree.  The second string represents an inorder traversal of a binary tree.  Important:  These are not search trees.  The labels/letters are there just to name the vertices.  They need not be placed in the tree in any order.  Look at the second example below.  The C is to the left of B in the tree.

Output:

Each such input line uniquely characterizes a binary tree.  For each such line, you must write onto the output the sequence of letters that would result from a postorder traversal of the binary tree.

End of Input:

The input will be terminated by a blank line.  Once this blank line is reached on the input, simply output one more blank line on the output and then immediately return control to the caller (i.e. exit from the run function).

Example:

For example, if the input coming in on the Buffered Reader looked like:

DBACEGF ABCDEFG
BCAD CBAD

the output must look like:

ACBFGED
CDAB

Note that the input contains 3 lines total.  (There is a blank line following "BCAD CBAD".)  The output also contains 3 lines. (There is a blank line following "CDAB".)  Once again, it is vital that you get the format exactly correct as the testing will be automated and will require this format.  Also, you may assume that in each problem, a letter does not appear more than once.  This means that the tree cannot have more than 26 nodes total.

In case some students are rusty on tree traversals, note that the first line of input corresponds to the tree:

                                    D
                                   / \
                                  /   \
                                 B     E
                                / \     \
                               /   \     \
                              A     C     G
                                         /
                                        /
                                       F

while the second line of input corresponds to the tree:

                                    B
                                   / \
                                  /   \
                                 C     A
                                        \
                                         \
                                          D
 

As always, the actual sequence of events interleaves the input and output.  After reading a line of input (which is an entire problem), you cannot read any more input until you print out the postorder traversal of the tree to the output.
