Problem #5

Write a public static function run in a class called Solution that takes two arguments, a BufferedReader from which to read inputs and a PrintWriter on which to write output, and which may throw an IOException.  You should use the skeleton provided under course documents on COL. You may optionally design/create other classes as well.

You will also need the test client which you can download from course documents on COL.

To run the test, make sure TestClient.java and your Solution.java are in the same directory and compile them.  You can then run the test at the command line with:

    java TestClient uname num

where uname consists of the first letter of your first name followed by the first seven letters of your last name, and num is the problem number.  For example, if David Letterman were submitting a solution to problem #5, he would type in

    java TestClient dletterm 5

at the command line.

Once the grading server is up and running, you can check your grade at:  http://wmarrerowww.cstcis.cti.depaul.edu/csc421/problem5.html

Note:

This problem depends critically on having a solution to problem 4.  Once the late period for problem 4 has expired look here for a link to a compiled solution to problem 4:  http://wmarrerowww.cstcis.cti.depaul.edu/csc421/Merge.class.

This class will have a single static method called merge that will merge two silhouettes into a single silhouette.  You can see a formal javadoc API here:  http://wmarrerowww.cstcis.cti.depaul.edu/csc421/Merge.html

Input:

The input coming in on the BufferedReader consists of a list of buildings, one per line, followed by a blank line.  Each building (line) will consist of 3 positive integers separated by a spaces.  The first is the x coordinate of the left side of the building.  The second is the height of the building.  The third is the width of the building..  As usual, you must supply your answer to a problem before continuing on to the next problem, so your code must stop after reading the blank line which marks the end of the problem.  After answering the problem, your code should continue reading again to get the next problem.

Output:

As discussed in class, your output in reply to reading in a list of buildings is the single silhouette that would result from a scene containing all the buildings.  You output should consist of en even number of integers, one per line, followed by a blank line that will mark the end of your answer.

End of all Input:

The end of all input will be marked by an additional blank line after the blank line marking the end of the last problem.  Once this extra blank line is reached on the input, you may return control to the caller of your run function.

Hints:

  • You will need to use divide and conquer on this problem.  If you use the merge algorithm to iteratively add buildings into "the silhouette so far", your solution will timeout on the larger problems.
  • A valid silhouette has an even number of integers, has at least 4 integers, and ends in 0.  My Merge.merge() function will throw a RuntimeException if either argument violates these necessary conditions for a valid silhouette.
