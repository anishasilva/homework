2/15/2011

 

 

Since I primarily use unix based systems the whole windows thing was a bit of a mystery to me.  So instead of compiling and running all of the auxiliary java programs for testing through batch files I just placed xpp3_min-1.1.3.4.O.jar and xstream-1.2.1.jar in my classpath.  Interestingly enough this was a lot easier to do in windows than in unix.  For setting the classpath in windows I just had to edit it under the environment variables deep in control panel.  So with that I didn’t need the batch compile with including the .jar files every time.  I just needed a simple :

MyWebserver.java:

 

javac MyWebServer.java

java MyWebServer

 

Handler.java:

 

javac Handler.java

java -Dfirstarg=%1 Handler (when called from shim.bat)

 

BCClient.java:

 

javac BCClient.java

java BCClient

 

BCHandler.java:

 

javac BCHandler.java

java -Dfirstarg=%1 BCHandler (when called from shim.bat)

 

For the final complete project with the back channel working, it was as easy as:

1. java MyWebServer

2. navigate IE to http:/localhost:2540/mimer-data.xyz

3. watch shim.bat open and run start the BCHandler which gets the temp windows file then

4. watch the web server console spit out the data in XML and in the deXMLized version.
