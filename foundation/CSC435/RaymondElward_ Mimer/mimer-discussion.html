I had a lot of problems beginning with this program. Eventually I got everything working perfectly though.  All steps through mimer-call.html are now working perfectly.

Without any resources about how to get a batch type file to work on unix I had to go ahead and install windows.  Luckily I’m a depaul student and I get access to a free windows 7 professional license.  That took a while but once I got it set up with java and netbeans and textpad installed it went pretty smoothly.  Associating the mime type was much easier than the lecture and assignment sheet made it out to be. 

Following the instructions on http://www.annoyances.org/exec/show/article02-500 made life easy.  I got the mime type associating.  I ran MyWebServer (which happily ran on windows after never having tried it there) to test it passing the new mime type “application/xyz” and it worked perfectly.  shim.bat would run passing in a temp version of mimer-data.xyz.  I was happy and moved on.

Now I copied Handler.java so that it could read environment variables, call it from server, through shim.bat.  I wrote the client side java handler for .xyz data.  shim.bat was used here to call Handler.  One issue that sprang up here was I kept getting a weird runtime exception error when running the handler.  I took the question to the newsgroup and came back with the answer that solved my problem.  I hadn’t created the directory C:\temp\ for the handler program to write mimer.output to.  Once I created that folder everything ran correctly.  I commented nearly every line in the program, but left the header because I didn’t change any of the code.

This is where I went into the windows settings to set the classpath with xstream-1.2.1.jar and mimer/xpp3_min-1.1.3.4.O.jar .  I put them both in the c:\program files\java\jdk1.6.0_23\lib\ folder.  Then I changed the windows environment variables so that the class path now also pointed to both of those jar files.  That way I wouldn’t have to compile and run while pointing to both those jar files every time I changed a little bit in the code and needed to recompile and retest.

After that I took the BCClient.java and commented out the portions that needed server interact for it to run.  I then compiled and ran it.  After typing in phrases I checked c:\temp\mimer.output .  everything was writing to this temp file on disk.  I also had it display on the console so I could see right away what the client was passing around.

 Then I used BCClient.java to setup back channel communication with MyWebServer.  This was pretty easy considering all the code was given to us.  Added the admin looper to my web server.  That was really easy considering I did a nearly identical admin type connection on the joke server project.  Then I copied in BCClient.java and made line by line comments on what the program was doing.  Then I started up the server, connected to it with the BCClient and made sure the output was showing correctly on both consoles.   

After everything was working perfectly individually I needed to connect it all together.  I created BCHandler by taking the XML marshalling capabilities and the client connection abilities from BCClient and then the capabilities to handle the name of the incoming temporary file, open it and read it from Handler.  Every step that was required of mimer-call.html I have completed and I understand it entirely.
